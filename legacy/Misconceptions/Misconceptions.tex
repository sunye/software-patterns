\begin{slide} {Première fausse idée}
\begin{verse}
	"Un pattern est la solution à un problème dans un contexte."
\end{verse}

Un contre-exemple simple:

\begin{description}
	\item[Probl\'eme:]  Comment récupérer mon ticket de loto avant son 
expiration?

	\item[Contexte:]   Le chien a avalé le ticket une heure auparavant.

	\item[Solution:]  Ouvrir le chien, retirer le ticket et courir au 
bureau de tabac le plus proche.
\end{description}
\end{slide}

 
\begin{slide} {Première fausse idée}
\begin{verse}
	"Un pattern est la solution à un problème dans un contexte."
\end{verse}

Que manque-t-il?
\begin{description}
	\item[R\'ecurrence:] La solution est pertinente ailleurs.
	\item[Enseignement:]  Le lecteur peut tailler la solution à son 
problème.
	\item[Nom:]  Une manière de faire référence au pattern.
\end{description}
\end{slide}

\begin{slide} {Un vrai pattern}
\vspace{1cm}
\begin{description}
	\item[Nom]  Generation Gap.

	\item[Contexte]  Un outil qui génère du code automatiquement. Les 
	programmeurs modifient le code pour ajouter ou modifier le 
	comportement.
\end{description}
\bigskip
\begin{verse}
	Comment empêcher que la régénération subséquent efface les 
	modifications du programmeur?
\end{verse}
\end{slide}



\begin{slide} {Un vrai pattern}
\vspace{1cm}
Forces:
\begin{itemize}
	\item  Besoin d'accès contrôlé au code généré.
	\item  Une approche par commentaires est  peu esthétique et non vérifiable.
	\item  Trouver les différences et fusionner automatiquement est trop 
incertain.
\end{itemize}
\end{slide}


\begin{slide} {Un vrai pattern}
\vspace{1cm}
Solution:
\begin{itemize}
	\item Utiliser l'héritage pour séparer le code généré des 
	modifications. 
\end{itemize}

\includegraphics [width=6cm] {generation-gap.eps}
%diagramme de classes

\end{slide}


\begin{slide} {Un vrai pattern}
\vspace{1cm}
Conséquences:
\begin{itemize}
	\item  Les modifications sont séparées du code générée.

	\item  Les modifications peuvent avoir un accès privilégié.

	\item  La régénération n'implique pas la remodification du code.

	\item  Le nombre de classes est doublé.

	\item  L'intégration dans des hiérarchies de classe existantes est 
	plus complexe.
\end{itemize}
\end{slide}

\begin{slide} {Résumé}

    Les patterns ne sont pas:
\begin{itemize}

\part{Fausses idées sur ce que les patterns peuvent faire.}

\begin{slide} {Cinquième fausse idée}
\vspace{1cm}
\begin{verse}
        "Les patterns assurent la réutilisabilité de code, meilleure 
        productivité, la paix dans le monde, etc."
\end{verse}
\vspace{1cm}
Les patterns n'assurent rien du tout! 

Ils donnent le pouvoir de le faire \--- potentiellement.
\end{slide}


\begin{slide} {Sixième fausse idée}
\vspace{1cm}    
	\begin{verse}
        "Les patterns \emph{génèrent} des architectures complètes."
	\end{verse}
	\vspace{0.5cm}
    	Similaire à la fausse idée précédente, mais plus prétentieuse.
\end{slide}


\begin{slide} {Qu'est-ce que la "générativité"?}
\vspace{1cm}
Il n'y a pas de consensus. \\
Le point de vue de Vlissides:
\begin{itemize}
	\item La capacité d'un pattern de créer un ``comportement émergent''.
	\item Elle fait qu'un pattern devienne plus qu'une solution rapide.
	\item Se repose sur les patterns.
\end{itemize}
\medskip
\emph{Les patterns ne génèrent rien du tout, les gens, oui!}
\end{slide}

\begin{slide} {Huitième fausse idée}
\vspace{1cm}
	\begin{verse}
	«Il n'y a pas d'évidence que les patterns peuvent aider quelqu'un.»
	\end{verse}

	Les bénéfices sont décrits dans:
	\begin{itemize}
	\item des revues (CACM, SP\&E).
	\item des conférences (OOPSLA, ECOOP, ICSE).
	\item des listes de discussion (patterns-discussion@cs.uiuc.edu)
	\end{itemize}

	\medskip
	\emph{L'évidence est plutôt qualitative.}
\end{slide}


\begin{slide} {Résumé}
\vspace{1cm}
	Les patterns ne peuvent pas:
	\medskip
	\begin{itemize}
	\item assurer la qualité, ni la réutilisabilité, ni rien d'autre.
	\item générer des architectures entières.
	\item être limités à la conception à objets.
	\item être écartés en tant qu'immatures.
	\end{itemize}
\end{slide}

\begin{slide} {Neuvième fausse idée}
	\begin{verse}
	``La communauté des patterns est une clique d'élites
	\end{verse}	

	\begin{description}
	\item [\'elite:] élite  n.  f. 1.  Ensemble  formé  par  les  
meilleurs  éléments  d'une  communauté.  Oeuvre  destinée  à  une  
élite. D'élite:  parmi  les  meilleurs.  \--- Un  tireur  d'élite,  
particulièrement  habile. 2.  (Plur.)  Les  élites:  les  membres  
des  catégories  sociales  jouissant  d'une  position  
particulièrement  élevée.
	\item [clique:] n.f. 1.  Péjor.  Groupe,  coterie.  Clique  de  
politiciens  véreux. 2.  MILIT  Ensemble  des  tambours  et  des  
clairons  d'un  régiment. 3.  SOCIOL  Groupe  de  personnes  liées  
par  des  obligations  mutuelles.
	\end{description}
\end{slide}
